# worker-go-http 日记 2026-06-05

## T-3: HTTP Layer 完成

### 完成内容

- **5 个 middleware**：JWT（含 claims 提取 user_id/role）、APIKey（提取 X-API-Key + X-Business-Code）、CORS（包含预检 OPTIONS 处理）、Logging（请求日志含耗时）、Recovery（gin 内置）
- **6 个 handler**：Business（CRUD）、Worker（heartbeat + list）、Lock（acquire/renew/release/list，LockHeldError -> 409）、Playbook（create/search/get）、Event（append/list + SSE streaming）
- **Router**：admin 组（JWT middleware）6 个端点 + worker 组（APIKey middleware）9 个端点，共 15 个端点
- **server.go 重写**：从空骨架改为完整的 gin.Engine 组装，集成 CORS/Logging/Recovery + health check + RegisterRoutes

### 关键设计

- LockHeldError 使用 `errors.As` 判断，返回 409 + holder_worker_id
- SSE events 使用 `text/event-stream` + `gin.Flusher` 流式推送
- Router 通过 `cfg.JWTSecret` 和 `cfg.CORSOrigins` 读取配置
- 接受 4 个参数：`(*gin.Engine, *middleware.Middleware, *handler.Handler, *config.Config)`

### 技术笔记

- Go proxy 不可达，无法运行 `go mod tidy` 更新 go.sum，但所有 import 已与 go.mod 的 require 列表完全对齐
- gin 的 `gin.Recovery()` 自带 panic recover，无需手写
- 流式 handler 中 `c.Writer` 需要 cast 成 `gin.Flusher` 才能用 `Flush()`
